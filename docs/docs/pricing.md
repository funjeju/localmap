# Business: Pricing Model

> 가격, 결제 채널, 갱신 전략. 일회성 함정을 벗어나 학교의 1년 운영에 녹아드는 수익 구조.

## 1. 가격 철학

세 가지 원칙을 지킵니다.

1. **너무 싸면 의심받는다.** 학교 B2B에서 30만원은 “장난감” 신호. 200~300만원이 신뢰의 가격.
1. **잠재 사용자 200명짜리 도구다.** 학교당 학생 80~200명 + 교사 10~30명 + 학부모 노출. 학생 1명당 1년 1만원도 합리적.
1. **공공 무료를 가격 압력으로 받아들이지 않는다.** SGIS 에듀가 못 하는 UX·자동화·산출물·다국어로 가격을 정당화.

## 3. 결제 채널

### 3.1 S2B 학교장터 (한국 학교 메인)

- **장점**: 학교 행정 결제 인프라. 에듀파인 회계 시스템 연동. 한 번 등록되면 전국 학교에서 접근.
- **단점**: 입점 심사 + 카테고리 등록. 자동화 어려움.
- **운영 방식**:
  - 상품 단위로 등록 (Basic / Standard / Premium 각각).
  - 학교가 견적 요청 → 견적서 발급 → 학교 자체 결재 → 입금 → 라이선스 발급.

### 3.2 토스페이먼츠 (한국 개인)

- 개별 교사 카드 정기결제.
- 네이버페이, 카카오페이 연동.
- 학교 검토 전 시범 사용 진입점.

### 3.3 체더스 / 에듀테크 구매대행

- 개별 교사 단위 월 구독 가능 채널.
- 학교 예산이 아닌 교사 사비 또는 학교 비공식 결재.

### 3.4 Stripe (일본/관광/교회 글로벌)

- 카드, Apple Pay, Google Pay.
- 인보이스 결제 (B2B 대형 계약).
- SEPA, ACH (해외 확장 시).

### 3.5 직접 계약 (교육청 / 지자체)

- 광역 도입은 견적서 + 입찰 + 직접 계약.
- 법무팀 검토 필요.
- 연 단위 청구.

## 4. 결제 흐름 (시스템 관점)

```
[학교 구매 결정]
       │
       ▼
[견적 요청] → 영업 담당자 또는 자동 견적서 발급
       │
       ▼
[S2B 또는 직접 입금]
       │
       ▼
[Webhook 수신] → /api/billing/webhook/s2b
       │
       ▼
[Tenant 플랜 업그레이드]
  - plan: 'standard'
  - seatsAllowed: 30
  - billingAccountId 연결
       │
       ▼
[기능 플래그 활성화]
  - features: ['ai_description', 'parent_portal', ...]
       │
       ▼
[관리자 이메일 발송]
  - 영수증
  - 라이선스 키 (필요 시)
  - 다음 갱신일 안내
```

## 5. 갱신 전략 — 일회성 함정 벗어나기

### 5.1 다년 자산 락인

학교가 1년 누적한 핀 + 사진 + 음성 + 변천사는 갱신 안 하면 잃습니다. 데이터를 인질로 잡지는 않되, “다음 학년도에 이어서 쓰려면 갱신이 필요해요” 명확히 안내.

비활성 학교의 데이터는 **읽기 전용 1년 유지** → 이후 아카이브.

### 5.2 연간 자동 갱신

기본은 자동 갱신. 갱신 2개월 전 안내 메일.

### 5.3 갱신 인센티브

- 2년차 5% 할인
- 3년차 10% 할인
- 인근 학교 추천 시 다음 해 10% 추가 할인 (학교 풀 확장)

### 5.4 시즌별 결제 캠페인

- 1~2월: 다음 학년도 준비 시즌. 학교 예산 편성 시기. → 영업 집중.
- 8~9월: 2학기 시작 전. 추가 도입 캠페인.

### 5.5 학교 운영 통합 = 갱신 명분

`docs/features/export.md`에 명시된 월별 출력물 캘린더가 갱신의 근거가 됩니다. 1년 내내 행정 산출물이 자동으로 나오므로 학교가 “이거 끊을 수 없는데?” 상태가 됨.

## 6. 매출 시뮬레이션

### 6.1 보수적 시나리오 (12개월)

- 시범 학교 5곳 (무료) → 사례 확보
- Basic 학교 10곳 × 120만원 = 1,200만원
- Standard 학교 5곳 × 250만원 = 1,250만원
- 개별 교사 100명 × 99,000원 = 990만원
- **연 매출 약 3,440만원**

### 6.2 공격적 시나리오 (24개월)

- Standard 학교 100곳 × 250만원 = 2.5억원
- Premium 학교 10곳 × 480만원 = 4,800만원
- 교육청 1곳 (100교) × 100만원 = 1억원
- 개별 교사 1,000명 × 99,000원 = 9,900만원
- **연 매출 약 4.7억원**

### 6.3 빅사이즈 (36개월+)

- 광역 교육청 3곳 × 평균 2억원 = 6억원
- 일본 진출 학교 50곳 × 50만엔 = 2,500만엔 (약 2.3억원)
- 관광 지자체 5곳 × 평균 1,000만원 = 5,000만원
- **연 매출 10억원+ 가능 영역**

## 7. 인프라 마진 분석

학교 100곳 운영 기준:

|항목               |월 비용                          |
|-----------------|------------------------------|
|Vercel Pro + Blob|$40                           |
|Firebase Blaze   |$100~200                      |
|Claude API       |$50~150                       |
|Protomaps 호스팅    |$20                           |
|기타 (Stripe, 모니터링)|$30                           |
|**월 합계**         |**약 $250~450**                |
|연 환산             |**약 $3,000~5,500 (400~720만원)**|

학교 100곳 × Standard 250만원 = 매출 2.5억원 / 인프라 비용 720만원 → **인프라 마진 97%**.

인건비 별도. 영업/CS 인력 충원 후에도 마진 70%대 유지 가능한 SaaS.

## 8. 가격 변경 정책

- 기존 고객은 갱신 시점까지 기존 가격 유지.
- 가격 인상은 90일 전 사전 고지.
- 다운그레이드는 다음 갱신 시점부터.

## 9. 환불 정책

- 첫 30일 100% 환불 (학교 합리적 검토 기간).
- 이후 일할 환불 불가.
- 단, 서비스 중대 장애 시 일할 보상.
