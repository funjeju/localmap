# Operations: Roadmap

> Phase별 개발 일정, 마일스톤별 산출물, 인프라 비용, 인력 계획. 이 문서는 **3개월마다 갱신**한다.

## 1. 큰 그림

```
Phase 1: MVP                4주    시범 학교 1곳 첫 사용
Phase 2: 자동화 완성도       6주    시범 학교 5곳, 첫 유료 계약
Phase 3: 다국어·다도메인 기반 4주    일본·관광 진출 준비
Phase 4: 비즈니스 확장        12주+  교육청 광역, 일본 진출
                            ────────────
                            약 6~8개월에 매출 본격 발생
```

총 26주 (약 6개월) 후 매출 본격화가 현실적 목표.

## 2. Phase 1 — MVP (Week 1-4)

### 목표

**시범 학교 1곳이 실제 수업에 한 번 써본다.**

### 산출물

- 학교 등록 + 백지도 렌더 + 수동 핀 CRUD + 카테고리 토글
- 교사 인증 (이메일/Google)
- 학생 임시 코드 로그인
- 데스크탑 + 태블릿 반응형

### 주별 마일스톤

**Week 1: 기반 셋업**

- [ ] Next.js + Vercel + Firebase 프로젝트 초기화
- [ ] shadcn/ui 설치 + 디자인 토큰 정의
- [ ] Firebase Auth + Firestore Security Rules 기본
- [ ] CI/CD (Vercel + GitHub Actions)
- [ ] 환경변수 + 시크릿 관리

**Week 2: 지도 + 데이터 모델**

- [ ] Protomaps `.pmtiles` 한국 영역 셋업 → Vercel Blob
- [ ] MapLibre GL JS 통합, 백지도 스타일 JSON 작성
- [ ] Firestore 컬렉션 스키마 적용 (tenants/layers/pins)
- [ ] GeoHash 헬퍼 (`geofire-common`)

**Week 3: 핵심 CRUD**

- [ ] Tenant 등록 폼 + Kakao 지오코딩
- [ ] 시스템 레이어 6종 자동 생성
- [ ] 핀 추가/수정/삭제 + 실시간 리스너
- [ ] 카테고리 레이어 토글
- [ ] 교사 로그인 + 멤버십

**Week 4: 학생 모드 + 시범 운영**

- [ ] 학생 임시 코드 로그인
- [ ] 학생 모드 UI (큰 버튼, 단순화)
- [ ] 학생 입력 검증 큐
- [ ] 사진 업로드
- [ ] 시범 학교 1곳 온보딩 + 첫 수업

### 의도적으로 제외

- 자동 시드 (수동 핀 추가만)
- AI 기능
- PDF 출력
- 학부모 포털
- 다국어 (한국어만)
- 결제 (수동 트라이얼만)

이 단계는 **“동작하는 골격”** 확인이 목적.

## 8. 인력 계획

### Phase 1-2 (Month 1-3)

- 풀스택 개발자 1명 (창업자 본인 또는 공동 창업자)
- 시범 학교 영업/CS는 창업자 직접

### Phase 3 (Month 4-6)

- 풀스택 개발자 1명 추가 (i18n + 벡엔드 강화)
- 디자이너 파트타임 (UI 다듬기)
- 영업 1명 (교사 출신 권장)

### Phase 4 (Month 7-12)

- 백엔드 개발자 1명 (Firebase + 시드 파이프라인)
- 프론트엔드 개발자 1명 (다국어 UI + PDF)
- 영업 1명 추가 (교육청 + 일본 담당)
- CS 1명 (학교 응대)

### Year 2

- 프로덕트 매니저 1명
- 일본 현지 영업 1명
- 데이터/AI 엔지니어 1명

총 7~10명 규모로 ARR 10억원~ 운영 가능.

## 10. 의사결정 체크포인트

다음 시점에 명시적으로 멈춰서 다음 단계 진행 여부 결정.

- **Week 4 종료**: 시범 학교가 한 번이라도 끝까지 썼는가? → No면 UX 재설계.
- **Week 10 종료**: 유료 학교 3곳 이상 확보했는가? → No면 Phase 3 늦추고 영업 집중.
- **Week 14 종료**: 일본 + 관광 진출 코드 기반이 정말 완성됐는가? → 통합 테스트 필수.
- **Month 12 종료**: ARR 5,000만원 달성? → 투자/대출/긴축 결정.

## 11. 문서 갱신 정책

- 이 문서는 **3개월마다 갱신**.
- Phase 종료 시점에 회고 + 다음 Phase 재계획.
- 모든 마일스톤 완료 여부는 PR 또는 이슈로 추적.
- KPI는 매월 1일 측정 후 이 문서 8번 항목에 기록.
