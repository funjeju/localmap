# Business: Internationalization (i18n)

> 다국어는 일본 진출의 전제 조건이자, 향후 글로벌 확장의 기반. **처음부터 박아둬야 한다.** 1년 후에 “이제 일본도 해볼까” 하면서 시작하면 코드를 절반은 갈아엎어야 한다.

## 1. 지원 언어

|코드  |언어     |상태         |우선순위|
|----|-------|-----------|----|
|`ko`|한국어    |메인         |1   |
|`ja`|日本語    |일본 진출용     |2   |
|`en`|English|향후 글로벌 + 폴백|3   |

향후 추가 가능성: `zh-CN`, `zh-TW`, `vi`, `id`. 단, **추가 비용 = 번역 + QA + 시드 어댑터** 이므로 신중히.

## 2. 두 종류의 다국어 처리

다국어는 **UI 텍스트**와 **데이터 콘텐츠** 두 영역으로 나뉘며, 처리 방식이 다르다.

### 2.1 UI 텍스트 → next-intl

버튼 라벨, 메뉴, 에러 메시지 등 코드에 박힌 모든 문자열.

```
/messages/
├── ko.json
├── ja.json
└── en.json
```

### 2.2 데이터 콘텐츠 → LocalizedText 객체

핀 이름, 설명, Tenant 이름 등 Firestore에 저장되는 사용자 데이터.

```typescript
interface LocalizedText {
  ko?: string;
  ja?: string;
  en?: string;
}
```

## 3. next-intl 구조

### 3.1 라우팅

App Router의 `[locale]` 동적 세그먼트.

```
/app/[locale]/
├── (public)/
├── (auth)/
└── (tenant)/[tenantId]/
```

URL 예시:

- `/ko/seoul_hakdong_es/map`
- `/ja/tokyo_minato_es/map`

### 3.2 미들웨어

```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';

export default createMiddleware({
  locales: ['ko', 'ja', 'en'],
  defaultLocale: 'ko',
  localePrefix: 'always'    // /ko/... 강제 표기
});

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)']
};
```

### 3.4 컴포넌트 사용

```tsx
// Server Component
import { getTranslations } from 'next-intl/server';

export default async function Page() {
  const t = await getTranslations('tenant.create');
  return <h1>{t('title')}</h1>;
}

// Client Component
'use client';
import { useTranslations } from 'next-intl';

export function PinList({ pins }) {
  const t = useTranslations('pin');
  return <p>{t('count', { count: pins.length })}</p>;
}
```

### 3.5 절대 금지 사항

- 컴포넌트에 한국어/일본어 직접 하드코딩.
- 조건문으로 언어 분기 (`locale === 'ko' ? '저장' : '保存'` 같은 코드 금지).
- 번역 키 동적 생성 (`t(\`status.${value}`)`는 허용, 하지만 가능한 정적 키 권장).

PR 리뷰 시 위 사항 자동 lint 권장 (`eslint-plugin-i18n-text`).

## 4. LocalizedText 처리

### 4.1 저장 시 규칙

사용자가 한국어로만 입력한 경우:

```typescript
// 저장
{ name: { ko: "학동주민센터" } }

// ja, en은 빈 상태 (누락)
```

Premium 플랜이면 저장 후 Cloud Function이 자동으로 `ai/translate` 호출하여 누락 locale 채움.

### 4.2 표시 시 폴백 규칙

```typescript
// lib/i18n/localizedText.ts
export function pickLocalized(
  text: LocalizedText | undefined,
  userLocale: string,
  tenantLocale?: string
): string {
  if (!text) return '';
  
  // 1. 사용자 locale
  if (text[userLocale]) return text[userLocale];
  
  // 2. Tenant locale (기본 언어)
  if (tenantLocale && text[tenantLocale]) return text[tenantLocale];
  
  // 3. 정해진 우선순위로 폴백
  for (const fallback of ['ko', 'en', 'ja']) {
    if (text[fallback]) return text[fallback];
  }
  
  return '';
}
```

**원칙: 이 헬퍼만 사용. 컴포넌트에서 직접 `text.ko` 접근 금지.**

### 4.3 누락 시 시각적 표시

번역이 없으면 옅은 글씨 + ✨ 아이콘으로 “AI 번역 가능” 안내. 클릭 시 즉시 번역.

## 5. 폰트 처리

언어별 폰트가 섞이면 일관성이 깨진다.

|언어   |폰트              |라이센스        |
|-----|----------------|------------|
|ko   |**Pretendard**  |SIL OFL (자유)|
|ja   |**Noto Sans JP**|SIL OFL     |
|en   |**Inter**       |SIL OFL     |
|통합 폴백|Noto Sans (CJK) |SIL OFL     |

### 5.1 Tailwind 설정

```ts
// tailwind.config.ts
fontFamily: {
  sans: ['var(--font-pretendard)', 'var(--font-noto-sans-jp)', 'var(--font-inter)', 'system-ui'],
  ko: ['var(--font-pretendard)'],
  ja: ['var(--font-noto-sans-jp)'],
  en: ['var(--font-inter)'],
}
```

### 5.2 Locale 기반 자동 분기

`[locale]` 세그먼트 진입 시 `<body>`에 동적 폰트 클래스 부여.

```tsx
export default function LocaleLayout({ children, params: { locale } }) {
  return (
    <html lang={locale} className={fontClassFor(locale)}>
      <body>{children}</body>
    </html>
  );
}
```

### 5.3 PDF 출력 시 폰트

`@react-pdf/renderer`는 폰트를 별도 등록 필요.

```typescript
import { Font } from '@react-pdf/renderer';

Font.register({
  family: 'Pretendard',
  fonts: [
    { src: '/fonts/Pretendard-Regular.ttf' },
    { src: '/fonts/Pretendard-Bold.ttf', fontWeight: 700 },
  ]
});
```

다국어 혼용 PDF (예: 한국어 + 일본어)는 각 텍스트 노드 단위로 `fontFamily` 자동 분기.

## 6. 일본 진출 — Locale 어댑터

`tenant.locale = 'ja-JP'` 일 때 자동 분기되는 영역.

### 6.1 지오코딩 어댑터

- 한국: Kakao Local
- 일본: GSI 지명검색

상세는 `docs/integrations/external-apis.md`.

### 6.2 시드 어댑터

- 한국: 행안부 / 문화재청 / 어린이보호구역
- 일본: KSJ / e-Stat / 文化財オンライン

상세는 `docs/integrations/seed-adapters.md`.

### 6.3 주소 포맷

```typescript
// lib/i18n/address.ts
export function formatAddress(addr: AddressParts, locale: string): string {
  if (locale === 'ko') return `${addr.city} ${addr.district} ${addr.street} ${addr.detail}`;
  if (locale === 'ja') return `${addr.prefecture}${addr.city}${addr.ward}${addr.chome}-${addr.banchi}`;
  if (locale === 'en') return `${addr.street}, ${addr.city}, ${addr.country}`;
  return addr.raw;
}
```

### 6.4 날짜/시간 포맷

`date-fns` + locale별 import.

```typescript
import { format } from 'date-fns';
import { ko, ja, enUS } from 'date-fns/locale';

const LOCALE_MAP = { ko, ja, en: enUS };

formatDate(d, 'PPP', { locale: LOCALE_MAP[userLocale] });
// ko: "2026년 5월 22일"
// ja: "2026年5月22日"
// en: "May 22, 2026"
```

### 6.5 통화 포맷

`Intl.NumberFormat` 사용.

- ko: ₩2,500,000
- ja: ¥250,000
- en: $1,899

가격 표시는 항상 사용자 locale의 통화로. 단, 실제 청구는 계약 통화.

### 6.6 학년 시스템 차이

|항목    |한국       |일본             |
|------|---------|---------------|
|학기 시작 |3월       |4월             |
|초등 학년 |1~6학년    |1~6年生          |
|학년도 표기|“2026학년도”|“令和8年度” (서력 병기)|

`tenant.locale`에 따라 자동 분기되는 캘린더 모듈 필요.

```typescript
// lib/calendar/schoolYear.ts
export function currentSchoolYear(locale: string, date = new Date()) {
  if (locale === 'ko') {
    const m = date.getMonth();
    return m >= 2 ? date.getFullYear() : date.getFullYear() - 1;
  }
  if (locale === 'ja') {
    const m = date.getMonth();
    return m >= 3 ? date.getFullYear() : date.getFullYear() - 1;
  }
  return date.getFullYear();
}
```

## 7. 개인정보보호 차이

### 7.1 한국 PIPA

- 만 14세 미만 → 법정대리인 동의 필요.
- 학교가 수집하면 학교가 수탁자, 플랫폼은 처리자.
- 개인정보처리방침 한국어 의무.

### 7.2 일본 개정 個人情報保護法 (2022~)

- 16세 미만 → 보호자 동의 권장.
- 외국 사업자도 일본 거주자 데이터 처리 시 법 적용.
- データ越境 (국외 이전) 시 별도 동의.

### 7.3 데이터 리전 분리

일본 Tenant 데이터는 **Firebase asia-northeast1 (도쿄)** 리전에 저장. 별도 Firestore 인스턴스 고려.

```typescript
// lib/firebase/byLocale.ts
const FIREBASE_INSTANCES = {
  'ko-KR': firebaseAppKR,
  'ja-JP': firebaseAppJP,
};

export function dbFor(tenant: Tenant) {
  return getFirestore(FIREBASE_INSTANCES[tenant.locale] ?? firebaseAppKR);
}
```

**원칙: 모든 Firestore 접근은 `dbFor(tenant)` 경유.** 기본 `db` 사용 금지.

### 7.4 개인정보처리방침 다국어

`/privacy/{ko,ja,en}.md`. 법무 검토 후 게시. Locale 진입 시 자동 표시.

## 8. SEO & hreflang

공개 페이지(학부모 포털 등)는 hreflang 태그.

```tsx
<head>
  <link rel="alternate" hrefLang="ko" href="https://app.com/ko/..." />
  <link rel="alternate" hrefLang="ja" href="https://app.com/ja/..." />
  <link rel="alternate" hrefLang="x-default" href="https://app.com/ko/..." />
</head>
```

## 9. 번역 운영

### 9.1 메시지 카탈로그 관리

- `messages/ko.json`이 원본.
- 신규 키 추가 시 `ko`만 작성하고 빌드 타임 검증.
- `ja`, `en`은 번역 도구 (DeepL Pro + 사람 QA) 경유.

### 9.2 빌드 타임 검증

```bash
# scripts/validate-i18n.ts
# - ko 기준 누락 키 검사
# - ICU 플레이스홀더 일치 검사
# - 키 알파벳 정렬 검사
```

CI에서 실패 시 머지 차단.

### 9.3 번역 도구

- 단순 번역: DeepL Pro (월 €30~).
- 사용자 데이터 번역: Claude API (서버 사이드 호출).
- 인간 번역가: 일본 진출 직전 정식 QA 패스.

### 9.4 용어집

브랜드 핵심 용어는 번역 고정.

|ko   |ja     |en               |
|-----|-------|-----------------|
|우리 동네|わたしたちの町|Our Neighborhood |
|핀    |ピン     |Pin              |
|백지도  |白地図    |Blank Map        |
|변천사  |移り変わり  |Changes Over Time|

용어집은 `messages/_glossary.md`에 관리.

## 10. 테스트 전략

- 각 컴포넌트는 ko / ja / en 모두 렌더 가능해야 함.
- E2E 테스트는 locale별로 실행.
- 비주얼 리그레션 테스트 (Chromatic 등)로 폰트 깨짐 감지.

## 11. 분석 이벤트

- `i18n.localeChanged` (props: from, to)
- `i18n.translationMissing` (props: key, locale) — 누락 키 모니터링
- `i18n.fallbackUsed` (props: requested, used)
