# 로컬맵(LocalMap) 프론트엔드 UI/UX 구현 명세서

본 문서는 'LocalMap' 대시보드 디자인을 실제 웹 서비스로 구현하기 위한 마크다운(Markdown) 기반의 컴포넌트 및 레이아웃 명세서입니다. 교육용 하이퍼로컬 DaaS(Data as a Service) 생태계를 구축하기 위한 프론트엔드 요소들이 모두 포함되어 있습니다.

---

## 1. 전역 레이아웃 (Global Layout)
* **그리드 시스템:** 12-column 그리드 기반의 반응형 레이아웃 (Desktop 최적화)
* **배경/테마:** 연한 그레이 배경(`#F9FAFB` 추정)을 사용하여 화이트 카드 컴포넌트들이 돋보이도록 구성. 라운드 모서리(Border-radius)를 적극 활용하여 학생/일반 사용자에게 부드럽고 친근한 인상 부여.

---

## 2. GNB (Global Navigation Bar)
상단 고정(Sticky) 헤더로, 다크 네이비 배경을 사용하여 메인 비주얼과 자연스럽게 연결됩니다.
* **Logo:** 좌측 'LocalMap' 텍스트 및 심볼 마크
* **Menu:** 탐방하기 | 지도 | 지역정보 | AI 학습 | 체험학습 | 커뮤니티 (Hover 시 하이라이트 효과)
* **Search:** 우측 중앙 위치. `검색 (지역, 학교, 명소)` 플레이스홀더 포함. (입력 시 자동완성 드롭다운 연결)
* **User Utilities:** 종 모양(알림) 아이콘, 사용자 프로필 이미지

---

## 3. 메인 히어로 섹션 (Hero Section)
* **Visual Background:** 자연 풍경(들판, 산, 하늘)에 문(Door)이 열려 있는 초현실적이고 감성적인 합성 이미지 적용. 
* **Typography:**
    * Headline: "지도를 넘어, 경험으로 배우는 우리 동네 이야기" (Bold, White)
    * Sub-copy: "학교 주변의 모든 것을 탐험하고, AI와 함께 더 깊이 이해하세요."
* **CTA Buttons:** * `[탐방 시작하기]` (다크 네이비 Fill)
    * `[AI 학습 체험하기]` (투명 배경, White Outline)
* **Floating Feature Cards (4개):** 히어로 섹션 하단에 걸쳐서 떠 있는 형태(Negative Margin 활용).
    * 지도 기반 탐방 / AI 학습 도우미 / 체험학습 기록 / 지역 정보 허브
    * 아이콘 + 타이틀 + 짧은 설명문 구조

---

## 4. 메인 콘텐츠 컴포넌트 (Grid Areas)

### 4.1. 탐방 지도 위젯 (Interactive Map)
* **위치:** 우측 상단 큰 영역 차지
* **상단 컨트롤:** 지역 필터 (예: `서울특별시 강남구`), 장소 검색창
* **지도 영역:**
    * 좌측 사이드바: 전체, 학교, 명소, 자연, 맛집, 문화 아이콘 필터
    * 오픈스트리트맵(OSM) 기반의 심플한 커스텀 타일셋 적용 추정
    * 마커: 카테고리별 컬러 마커 (학교, 자연, 명소 등)
    * **Info Window (팝업):** 마커 클릭 시 등장. '역삼초등학교' (탐방 128 / 리뷰 56), `[상세보기]`, `[탐방하기]` 액션 버튼 포함.
    * 우측 컨트롤: 줌인/아웃, 내 위치, 지도 스타일 변경 버튼

### 4.2. 탐방 추천 갤러리 (Explore Recommendations)
* **위치:** 히어로 섹션 바로 아래 좌측 영역
* **구조:** 수평 스크롤이 가능한 카드 리스트 (캐러셀 슬라이더)
* **탭 메뉴:** 전체 | 학교 주변 | 역사 | 자연 | 문화 | 맛집
* **장소 카드:** * 장소 이미지 (상단)
    * 뱃지 (예: `학교`, `명소`, `역사`)
    * 타이틀 및 한 줄 설명 ("역삼초등학교 - 우리 학교에 대해 더 자세히 알아보아요")
    * 별점(4.8) 및 탐방 횟수 표기

### 4.3. AI 요약 및 추천 코스 (우측 사이드바)
* **오늘의 AI 요약:** 특정 장소(예: 역삼동)의 역사적 배경을 AI가 요약해준 텍스트 카드. `[전체 요약 보기]` 버튼.
* **추천 탐방 코스:** 리스트형 UI. (역사 탐방 코스 3.2km, 자연 관찰 코스 등). 클릭 시 지도의 동선 레이어로 연결.

### 4.4. 게미피케이션 및 활동 패널 (Gamification & Activity)
하단은 사용자 참여를 유도하는 데이터 위젯들로 구성됩니다.
* **나의 탐방 현황 (좌측):** * 원형 프로그레스 바 (75%)
    * 레벨 텍스트 ("Lv. 3 탐험가", "다음 레벨까지 250 XP")
    * 통계 지표: 탐방 완료(24), 리뷰 작성(18), 학습 자료(12), 뱃지 획득(8)
* **이번 주 탐방 챌린지 (우측 하단):**
    * 다크 그린 배경의 포인트 카드
    * 미션: "5곳 탐방하기" (프로그레스 바 3/5)
    * 보상: 100 XP, 방패 모양 뱃지 아이콘

### 4.5. 데이터 연동 컴포넌트 (타임라인, AI 학습, 최근 활동)
* **지역 타임라인:** 수직형 타임라인 UI. 연도별 이벤트(2024년, 1990년대, 조선시대)를 나열.
* **AI 학습 도우미:** "새 학습 자료 만들기" 버튼 및 최근 생성된 AI 맞춤형 학습 자료 리스트 (보기 버튼 포함).
* **최근 활동 (UGC 실시간 피드):** 프로필 이미지 + "김학생님이 역삼초등학교를 탐방했어요" 형태의 실시간 로그 피드.

### 4.6. 체험학습 보고서 생성 배너
* 풍경 일러스트 배경을 가진 가로형 와이드 배너.
* "탐방 내용을 정리하고 멋진 보고서를 만들어보세요" + `[보고서 만들기]` 버튼.

## 6. 구현 시 기술적 고려사항 (Technical Considerations)
1.  **맵 렌더링 엔진:** 지도 컴포넌트는 Mapbox GL JS 또는 React Three Fiber(R3F)를 활용해 마커 인터랙션과 커스텀 뷰를 렌더링합니다.
2.  **데이터 바인딩 (DaaS 구조):** '탐방 추천', '최근 활동', '타임라인'은 모두 Firebase 또는 Supabase와 같은 실시간 데이터베이스(Real-time DB) 리스너를 붙여 학생들이 올린 데이터(UGC)가 즉각 반영되도록 컴포넌트를 설계합니다.
3.  **컴포넌트 모듈화:** React/Next.js 환경에서 각 패널(카드, 맵, 프로그레스, 리스트)을 독립적인 컴포넌트로 분리하여 재사용성과 유지보수성을 극대화합니다.
