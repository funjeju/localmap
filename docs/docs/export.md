# Feature: Export

> “우리 동네 책” PDF, A3 포스터, 학교평가용 리포트. **갱신율을 만드는 가시적 산출물**의 영역.

## 1. 왜 출력이 중요한가

디지털 활동만으로는 1년 후 갱신 결제가 어렵습니다. 학년말에 손에 잡히는 산출물이 나와야:

- 학부모 공개수업/학예회에서 자랑할 수 있음 → 학교 가시성↑
- 교장/교감에게 “이런 결과물이 나왔어요” 보고 → 예산 갱신 정당화
- 학교평가 실적으로 제출 → 행정적 정당성

따라서 출력은 **단순 PDF 다운로드가 아니라 “학교 운영에 녹아드는 산출물”** 로 설계합니다.

## 2. 출력 종류

### 2.1 우리 동네 책 (Yearbook)

학년말 종합 PDF. 32~64페이지. 핀 데이터 + 학생 인터뷰 + 사진 + 변천사.

### 2.2 A3 포스터 (Poster)

교실 게시판/복도 게시용. 카테고리별 색상으로 핀 표시한 한 장짜리.

### 2.3 단원별 학습지 (Worksheet)

수업 차시에 맞춘 흑백 인쇄용. 백지도 + 빈칸 + 학습 활동.

### 2.4 학교평가 리포트 (Report)

행정용. 활동량, 참여 학생 수, 카테고리 분포, 변천사. 학교평가/교육청 보고용.

### 2.5 학부모 안내문 (Newsletter)

학기별 학부모 발송용 A4 1~2장. 우리 반이 만든 동네 지도 소개 + QR.

## 3. 우리 동네 책 — 상세 설계

### 3.1 페이지 구성

```
[표지]
  우리 동네 이야기 책
  2026학년도 3학년
  학동초등학교

[2-3p] 인사말 (교사 작성 또는 AI 초안)

[4-5p] 우리 동네 한눈에 보기
  - 학교 중심 백지도 + 모든 핀 표시
  - 카테고리별 통계

[6-15p] 공공기관 편
  - 카테고리 인덱스 페이지
  - 핀 1개당 1~2페이지 (사진 + 설명 + 학생 메모)

[16-25p] 명소 편
  ...

[26-35p] 옛이야기 편

[36-40p] 동네의 변화 (Timeline)
  - 1년간 새로 생긴/없어진 장소
  - 변경 이력 시각화

[41-45p] 우리 반 친구들이 만든 핀
  - 학급별 / 학생별 기여 모음

[뒷표지] 다음 학년도에게 — 인계 메시지
```

### 3.2 React-PDF 구조

```typescript
// lib/export/templates/yearbook.tsx
export const YearbookTemplate = ({ tenant, project, pins, layers }) => (
  <Document>

```

### 3.3 지도 렌더링

PDF에 지도 이미지가 들어가야 함. 두 가지 방법:

**A. 서버 사이드 MapLibre 렌더 (권장)**

- Cloud Function에서 Playwright/Puppeteer로 MapLibre 페이지 캡처.
- 고화질 PNG 추출 → React-PDF에 삽입.

**B. 정적 타일 합성**

- Protomaps의 정적 이미지 API.
- 더 빠르지만 핀 오버레이 별도 그리기 필요.

MVP는 A 방식.

### 3.4 디자인 토큰

PDF 디자인은 Tenant 브랜딩 반영.

- `tenant.branding.primaryColor`
- 폰트: 한국어 Pretendard, 일본어 Noto Sans JP, 영어 Inter.
- 한 책에 다국어 혼용 시 폰트 자동 분기.

## 5. 단원별 학습지

교과서 차시와 매칭.

### 5.1 3학년 1학기 사회 1단원용 예시

**1차시: 우리 고장의 장소**

- 백지도 인쇄용 (학교 중심, 흑백)
- 학생이 직접 핀을 그리고 색칠하는 빈칸

**2차시: 디지털 영상 지도와 비교**

- 디지털 핀이 표시된 지도 + 빈칸 메모란

**3차시: 백지도에 우리 고장 표시**

- 카테고리별 범례
- 학생이 골라서 표시할 빈 백지도

이런 차시별 템플릿을 미리 라이브러리로 보유.

## 6. 학교평가 리포트

### 6.1 포함 지표

- 활성 교사 수 / 학생 수
- 총 핀 수, 학생 입력 핀 수
- 카테고리 분포
- 월별 활동량 (그래프)
- 변경 이력 건수
- AI 사용량
- 학부모 포털 접근 수

### 6.2 형식

A4 5~10페이지. 차트는 Recharts → 서버 렌더링.

행정 보고용이므로 **공공누리 스타일의 단정한 디자인**.

## 7. 비동기 생성 인프라

### 7.1 작업 큐

```
[사용자] → POST /api/export/pdf
              │
              ▼
       exports/{exportId} 도큐먼트 생성 (status: queued)
              │
              ▼
       Cloud Function (onCreate) 트리거
              │
              ▼
       1. Firestore에서 핀/이력 데이터 페치
       2. 지도 이미지 렌더 (Playwright)
       3. React-PDF로 PDF 생성
       4. Firebase Storage 업로드
       5. exports/{exportId} 업데이트 (status: completed, downloadUrl)
              │
              ▼
[클라이언트] 폴링 또는 onSnapshot으로 완료 감지
```

### 7.2 소요 시간

- 우리 동네 책 (40p): 60~120초
- A3 포스터: 15~30초
- 리포트: 10~20초

사용자에게는 “1~2분 안에 준비됩니다. 완료되면 알려드릴게요” 안내.

### 7.3 캐싱

같은 (projectId, templateKey, locale)은 결과 캐싱. 다시 요청 시 즉시 반환. 데이터 변경 시 캐시 무효화.

### 7.4 비용

- Cloud Functions 메모리 2GB (Playwright 무거움).
- 초당 비용 약 $0.0001. 60초 작업 ≈ $0.006.
- 학교당 월 5권 PDF → 약 $0.03. 무시 가능.

## 8. 출력 라이프사이클 — 학교 1년 캘린더

|시기 |출력물          |트리거        |
|---|-------------|-----------|
|3월 |신학기 학부모 안내문  |자동         |
|4월 |사회 1단원 학습지   |교사 명시      |
|6월 |1학기 중간 리포트   |자동 (교장 발송) |
|9월 |2학기 학부모 안내문  |자동         |
|11월|학예회용 A3 포스터  |교사 명시      |
|12월|우리 동네 책 (학년말)|교사 명시      |
|1월 |학교평가용 리포트    |자동 (관리자 알림)|

자동 트리거는 Cloud Scheduler. 교사 알림 + 다운로드 링크.

|케이스            |처리                                          |
|---------------|--------------------------------------------|
|핀 데이터 0개       |“아직 핀이 없어요” 안내 페이지로 책 생성 거부                 |
|사진이 너무 많아 용량 초과|카테고리당 대표 사진 5장만 자동 선별                       |
|일본어 + 한국어 혼용   |폰트 자동 분기, 라인 높이 조정                          |
|학생 이름 비식별 처리 요청|옵션으로 “닉네임 모드” 지원                            |
|PDF 생성 중 실패    |자동 재시도 1회, 그래도 실패 시 교사 알림 + 부분 데이터로 단순 버전 제공|
